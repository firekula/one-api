# 用户手册

## 核心概念

在使用 One API 之前，了解以下几个核心概念会帮助你更好地理解系统的工作方式。

### 渠道 (Channel)

渠道是上游 API Key 的封装。一个渠道对应一个 AI 服务提供商的 API Key，包含以下信息：

- **类型**：如 OpenAI、Anthropic Claude、Google Gemini、百度文心一言等，目前支持 27+ 种模型提供商
- **Base URL**：API 请求的目标地址
- **API Key**：上游服务提供商的认证密钥
- **模型列表**：该渠道可使用的模型（如 `gpt-4o`、`gpt-4o-mini`）
- **分组**：用于与用户分组配合，控制谁可以使用该渠道

你可以为同一个模型提供商创建多个渠道（例如多个 OpenAI 账号），One API 会自动在它们之间进行负载均衡。

### 令牌 (Token)

令牌是分发给终端用户的 API Key，是终端用户调用模型时的凭证。每个令牌包含：

- **额度限制**：该令牌最大可消耗的额度，`-1` 表示不限制
- **过期时间**：令牌的有效期，`-1` 表示永不过期
- **IP 白名单**：允许使用该令牌的 IP 地址列表
- **模型访问控制**：该令牌可以调用哪些模型

令牌的格式为 `sk-xxxxxxxxxxxxx`，用户在客户端中将其设置为 OpenAI API Key 即可使用。

### 兑换码 (Redemption)

兑换码是预先生成的充值码。管理员可以批量生成兑换码并分发给用户，用户输入兑换码后即可为账户增加额度。常用于商业化场景或活动运营。

### 用户分组 (User Group)

用户归属于某个分组，默认分组为 `default`。用户分组与渠道分组配合，决定了用户可以访问哪些渠道。例如：

- 用户 A 属于 `vip` 分组
- 渠道 1 属于 `vip` 分组
- 渠道 2 属于 `default` 分组
- 那么用户 A 只能使用渠道 1，无法使用渠道 2

### 渠道分组 (Channel Group)

渠道归属于某个分组，默认分组为 `default`。只有与用户分组匹配的渠道才会被该用户访问到。

### 额度 (Quota)

额度是 One API 内部的计费单位，**不是真实货币**。系统通过额度的消耗来统计使用量，方便管理员进行计费管理。额度的计算公式见下文"额度规则"章节。

## Web 管理界面概览

登录管理面板后，左侧导航栏包含以下菜单项：

| 菜单 | 功能 | 可用角色 |
|------|------|----------|
| **首页** | 显示系统公告、使用统计、最近请求图表 | 管理员 / 普通用户 |
| **渠道** | 管理上游 API Key 和提供商配置 | 仅管理员 |
| **令牌** | 创建和管理 API 访问令牌 | 管理员 / 普通用户 |
| **兑换码** | 批量生成和管理充值兑换码 | 仅管理员 |
| **日志** | 查看 API 请求日志和消费记录 | 仅管理员 |
| **设置** | 系统全局配置 | 仅管理员 |
| **关于** | 系统版本信息、自定义展示内容 | 管理员 / 普通用户 |

**管理员与普通用户的区别：**

- **管理员**：可以看到所有用户、所有渠道、所有日志，拥有系统设置和兑换码管理权限。默认管理员账号为 `root`，初始密码 `123456`。
- **普通用户**：只能看到自己创建的令牌，无法查看渠道和日志，也无法访问系统设置页面。

## 操作指南

### 创建第一个渠道

以下以添加 OpenAI 渠道为例：

1. 登录管理面板，进入左侧导航栏的 **渠道** 页面
2. 点击 **添加渠道** 按钮
3. 填写以下信息：
   - **渠道名称**：输入一个便于识别的名称，如"我的 OpenAI 账号"
   - **渠道类型**：从下拉列表中选择 `OpenAI`
   - **API Key**：输入你的 OpenAI API Key（格式如 `sk-proj-xxxxxxxxxxxx`）
   - **Base URL**（可选）：留空使用默认值 `https://api.openai.com`，如需代理可填写自定义地址
4. 设置 **模型列表**：
   - 可以从下拉菜单中勾选需要的模型，如 `gpt-4o`、`gpt-4o-mini`
   - 也可以手动输入模型名称，多个模型用逗号或换行分隔
5. 设置 **分组**：默认分组为 `default`，后续可通过分组控制用户访问权限
6. 点击 **保存** 完成创建

创建完成后，可以在渠道列表看到该渠道。点击测试按钮可以验证渠道是否配置正确。

### 创建令牌

1. 进入左侧导航栏的 **令牌** 页面
2. 点击 **添加令牌** 按钮
3. 填写以下信息：
   - **名称**：输入便于识别的名称，如"我的令牌"
   - **额度**：输入最大可用额度。`-1` 表示不限制额度
   - **过期时间**：设置令牌的有效期。`-1` 表示永不过期
   - **IP 白名单**（可选）：设置允许使用该令牌的 IP 地址，多个 IP 用逗号分隔。留空表示不限制
   - **允许模型**（可选）：设置该令牌可以调用的模型，留空表示允许调用所有模型
4. 点击 **保存**

保存后，页面会显示生成的令牌密钥。**请立即复制并保存**，关闭页面后将无法再次查看完整密钥。

### 在客户端中使用

拿到令牌后，在任意 OpenAI API 兼容的客户端中设置以下信息：

```bash
OPENAI_API_BASE="https://your-domain.com/v1"
OPENAI_API_KEY="sk-xxxxxxxxxxxxx"
```

请将 `your-domain.com` 替换为你部署 One API 的实际域名或 IP 地址。

**常见客户端配置示例：**

**ChatGPT Next Web：**
1. 进入设置页面
2. 将 OpenAI API 地址设置为 `https://your-domain.com/v1`
3. 将 API Key 设置为 `sk-xxxxxxxxxxxxx`
4. 保存即可

**Cherry Studio：**
1. 进入设置页面
2. 选择添加自定义模型提供商
3. 填写 API Base URL 为 `https://your-domain.com/v1`
4. 填写 API Key 为 `sk-xxxxxxxxxxxxx`
5. 添加你需要的模型名称
6. 保存并开始使用

**Python OpenAI SDK：**

```python
import openai

openai.api_base = "https://your-domain.com/v1"
openai.api_key = "sk-xxxxxxxxxxxxx"

# 调用模型
response = openai.ChatCompletion.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)
```

**cURL：**

```bash
curl https://your-domain.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-xxxxxxxxxxxxx" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

### 指定渠道

在某些情况下，你可能希望强制使用某个特定渠道（而不是让 One API 自动负载均衡）。可以通过在令牌后添加渠道 ID 来实现：

```
Authorization: Bearer sk-xxx-1
```

其中 `1` 是渠道 ID，可以在渠道列表页面查看。

> **注意**：只有管理员创建的令牌才能指定渠道。普通用户创建的令牌不支持此功能。

## 额度规则

### 计算公式

消费额度的计算公式如下：

```
消费额度 = 分组倍率 × 模型倍率 × (prompt_tokens + completion_tokens × completion_ratio)
```

各参数说明：

| 参数 | 说明 |
|------|------|
| **分组倍率** | 用户分组倍率 × 渠道分组倍率 |
| **模型倍率** | 每个模型独立配置的倍率，可在模型设置中查看和修改 |
| **prompt_tokens** | 输入（提示）的 token 数量 |
| **completion_tokens** | 输出（生成）的 token 数量 |
| **completion_ratio** | 输出 token 的额外倍率系数，部分模型输出比输入更贵 |

### 分组倍率

分组倍率由两部分相乘得出：

- **用户分组倍率**：管理员在用户管理中为用户所属分组设置的倍率
- **渠道分组倍率**：管理员在渠道管理中为渠道所属分组设置的倍率

例如，用户分组倍率为 `2`，渠道分组倍率为 `1.5`，则分组倍率为 `2 × 1.5 = 3`。

### 模型倍率

每个模型都有自己的倍率，与 OpenAI 官方定价相对应。例如：

| 模型 | 倍率 |
|------|------|
| `gpt-4o` | 5 |
| `gpt-4o-mini` | 0.25 |
| `gpt-3.5-turbo` | 0.5 |

模型倍率可以在系统设置中的模型配置中查看和调整。

### 账户额度与令牌额度

- **账户额度**：整个账户的总可用额度，所有令牌共享此额度
- **令牌额度**：单个令牌的最大可用额度，作为令牌级别的上限

实际消耗时，同时受两者限制。无论哪个额度先用完，该令牌都将停止服务。

### Stream 与非 Stream

流式请求（Stream）和非流式请求的计费方式相同。但需要注意的是，部分模型提供商在 Stream 模式下不返回 usage 信息，此时 One API 会尝试通过 tiktoken 本地计算 token 数量。如果本地计算失败，额度将按估算值扣除。

## 兑换码

### 管理员操作

1. 进入 **兑换码** 页面
2. 点击 **批量生成**
3. 设置生成数量和每个兑换码的额度值
4. 点击生成，系统会自动创建指定数量的兑换码
5. 点击 **导出 CSV** 将兑换码列表导出为文件
6. 将兑换码分发给用户（可通过邮件、微信等渠道）

### 用户操作

1. 登录管理面板
2. 进入 **充值 (TopUp)** 页面
3. 在输入框中粘贴兑换码
4. 点击确认，额度会自动充值到账户中

## 用户管理

### 注册方式

One API 支持多种用户注册和认证方式：

| 方式 | 说明 |
|------|------|
| **邮箱注册** | 用户直接注册，可设置邮箱白名单限制可注册的邮箱域名 |
| **GitHub OAuth** | 通过 GitHub 账号登录，需在系统设置中配置 GitHub OAuth App |
| **飞书 OAuth** | 通过飞书账号登录，需在系统设置中配置飞书应用 |
| **OIDC** | 通过任意 OIDC 兼容的身份提供商登录 |
| **微信公众号** | 通过微信公众号扫码登录 |

邮箱白名单功能：在系统设置中配置允许注册的邮箱域名（如 `@company.com`），只有匹配该域名的邮箱才能成功注册。

### 分组设置

管理员可以在用户管理页面为每个用户分配分组。分组决定了用户可以使用哪些渠道。

修改用户分组：
1. 进入 **设置 > 用户管理**
2. 找到目标用户，点击编辑
3. 在"用户分组"字段中设置分组名称
4. 保存

### 手动充值

管理员可以通过以下方式为用户手动增加额度：

- **通过管理界面**：在用户管理页面找到目标用户，点击"充值"，输入充值额度
- **通过 API**：调用充值接口，传入用户 ID 和充值额度

## 系统设置

### 公告 (Notice)

在系统设置中可以配置公告内容，公告会显示在所有用户的首页上方。支持纯文本和 HTML 格式。

适用场景：发布维护通知、版本更新公告、使用规范提醒等。

### 充值链接 (TopUp Link)

配置外部支付页面的 URL。配置后，用户在首页会看到"充值"按钮，点击后跳转到你设置的支付页面。

One API 会自动在 URL 末尾追加用户信息参数，方便你的支付系统识别用户：

```
https://your-payment-page.com/topup?user_id=123&username=xxx
```

你需要在支付页面中读取这些参数，在用户完成支付后调用 One API 的充值接口。

### 新用户初始额度

可以通过环境变量或系统设置来配置新用户的初始额度。新用户注册成功后，系统会自动为该用户赋予初始额度。

### 首页/关于页自定义

系统设置支持对首页和关于页面进行自定义展示：

- **HTML**：支持直接嵌入 HTML 代码，可实现富文本展示
- **Markdown**：支持 Markdown 格式的文本内容
- **iframe 嵌入**：可以通过 iframe 嵌入外部页面

> **提示**：如果你需要展示使用说明、API 文档或其他帮助信息，推荐使用 Markdown 格式，简洁且易于维护。
